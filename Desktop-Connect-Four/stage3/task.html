Description

Did you notice any problems with the last stage? If you accidentally enter a letter instead of a number, the program will crash. That wouldn't make for a very fun game if every time a player makes a typo, you have to start over from the beginning. In this stage we will account for user error with the input. This is extremely important for any program that takes user input because humans can easily make small mistakes, and we want to be able to deal with them. In this stage, we will fix our code so that users can try again if they enter a non-number value or if they enter a number that does not correspond to the column number. We will also give the users another chance to enter a different column if they one they chose is already full.

Objectives

In this stage your program should:

  1. Be able to handle different kinds of user input without crashing.
  2. Tell the user that the value is invalid if they enter in a non-number value or a number that isn't 1-7.
  3. Tell the user that the column is full if they enter a column number that is already filled.
  4. Give the user a chance to enter in another value in both of these cases.
