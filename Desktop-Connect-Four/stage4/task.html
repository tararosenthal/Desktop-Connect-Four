Description

Finally, we are ready to add a second player to our game! Changing the game to support two players requires a lot of refactoring. We must add another type of piece to the board so we know which player placed it. In the physical game, they are different colors, but in this desktop version, we will use different letters. We also need to switch players between turns so each person can get a fair chance at winning the game. Finally, we need to change the win conditions in the game. Previously, the game was over when the board was filled. Now we will need to check after each turn to see if either player has connected four pieces in a row. We need to do this in three directions: horizontally, vertically, and diagonally. After this stage, you will have a fully functioning game you can play with a friend!

Objectives

In this stage, your program should:

  1. Switch turns between players after each move.
  2. Represent Player 1's moves with the letter 'O' and Player 2 with the letter 'X'.
  3. Give the current player a chance to re-do their move if invalid input is detected or column is already full.
  4. Check after each move if either player has connected four in a row vertically, horizontally, or diagonally.
  5. Once a winner has been detected, print the board one last time along with the words "Player X wins!", where X stands for 1 or 2.
